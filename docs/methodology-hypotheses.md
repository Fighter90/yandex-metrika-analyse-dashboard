# Методология гипотез

> **Любое изменение методологии фиксируется здесь** с обоснованием (см. антипаттерн в README §15).
> Реализация — `code/shared/src/validation.ts`; skill — `.claude/skills/hypothesis-check/SKILL.md`.

## Формат гипотезы

> «{subject} {action} {solution}, **если** {condition}»

- **subject** — ЦА (например, «слушатель подкаста»).
- **action** — «готов купить / готов заплатить за / переключится с / оставит контакт».
- **solution** — что предлагаем.
- **condition** — условие истинности (без него гипотеза неполная).

## Обязательные элементы (иначе не сохраняется)

1. **Скрытые допущения — ≥3**, покрывающие все три категории: `behavior`, `market`, `tech`.
2. **Способы проверки — ≥2** разных типа: `synthetic` / `live` / `quantitative` / `market`.
3. **ICE** (см. `methodology-ice.md`) — `Impact × Confidence × Ease`, у каждого фактора rationale.
4. **Светофор** — 🟢/🟡/🔴, каждый с конкретным порогом и метрикой.
5. **Дедлайн** проверки (в днях).

Валидация на трёх уровнях: UI блокирует сохранение, репозиторий отвергает (HTTP 422),
тесты это фиксируют. См. `validateHypothesis`.

## Decision Log замыкает цикл

Каждая проверенная гипотеза → запись `DL-NNN` (что проверяли, что узнали, **цитаты/данные**,
исход, next step). При сохранении статус гипотезы автоматически становится исходом. Шаблон —
`.claude/skills/decision-log/SKILL.md`.
