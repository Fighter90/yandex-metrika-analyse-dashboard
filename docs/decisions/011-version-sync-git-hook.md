# ADR-011 — Синхронизация версии в доках через git-хук без husky

- **Статус:** принято
- **Дата:** 2026-05-27
- **Контекст:** fix-prompt FINAL §9.4 требует, чтобы версия в шапках доков не расходилась с
  `package.json`, и pre-commit проверку этого. `CLAUDE.md`: новая зависимость — только через ADR.

## Решение

Маркеры версии в доках синхронизирует zero-dependency скрипт **`scripts/sync-versions.mjs`**
(чистый Node ESM, без TS-раннера). Pre-commit хук лежит в репозитории — **`.githooks/pre-commit`**,
включается через `git config core.hooksPath .githooks` из npm-скрипта `prepare` (выполняется на
`pnpm install`). **husky не добавляем.**

## Обоснование

- `node scripts/*.mjs` исполняется штатно (`"type": "module"`), без `tsx`/`ts-node` в корне и без
  новой зависимости — соответствует принципу «минимум стека» из ADR-001.
- `core.hooksPath` + commit-friendly `.githooks/` даёт версионируемый, общий для команды хук —
  тот же эффект, что husky, но без пакета и его postinstall-магии.
- Скрипт идемпотентен и имеет режим `--check` (используется в хуке): падает, если маркер версии
  отстал от `package.json`, с подсказкой `pnpm sync-versions`.

## Последствия

- Маркеры обновляются только в заранее заданных строках (`docs/architecture.md`, `CLAUDE.md`,
  `README.md`, `README.en.md`); таблицы CHANGELOG, ссылки на релизы и даты не трогаются.
- При бампе версии: `pnpm sync-versions` → коммит. Хук не даёт закоммитить рассинхрон.
- Скрипт исключён из coverage (одноразовая dev-утилита, как прочие `scripts/`/CLI).
