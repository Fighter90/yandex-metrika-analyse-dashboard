# Spec 014 — ГОСТ-отчёт с графиками-изображениями

- **Статус:** принято (реализуется в v2.8.0)
- **Дата:** 2026-05-28
- **Связано:** fix-prompt FINAL §6; ADR-002 (без Docker), ADR-006 (100% покрытие),
  ADR-008 (server-side рендер графиков, вариант A).

## Цель

Отчёт (DOCX и PDF) из неизменяемого `ReportSnapshot` должен соответствовать ГОСТ-оформлению и
содержать **ключевые графики как растровые изображения** (PNG), а не только текст и таблицы. Из
одного `snapshotId` DOCX и PDF дают **идентичный контент** (включая одни и те же изображения).

## Нефункциональные требования

- **Детерминизм:** в render-пути нет LLM и `Date.now()`. Опции графиков — чистые функции от
  снапшота; анимации ECharts отключены (`animation:false`); размер/`deviceScaleFactor` фиксированы.
  Один снапшот → одинаковые PNG (на одной машине; кросс-машинный побайтовый SHA — best-effort,
  см. ADR-008).
- **Анти-галлюцинация:** каждое число на графике выводится из тех же агрегатов снапшота, что и
  таблицы; график не вводит новых данных.
- **Деградация:** если Chrome/ECharts недоступны, `renderReportCharts` возвращает `{}` и отчёт
  строится в текстово-табличном виде (графики опускаются), а не падает.

## Структура отчёта (разделы)

1. Титульная страница (центр, ГОСТ-поля).
2. Содержание (оглавление).
3. Краткие итоги (KPI, заявка ≠ оплата).
4. Рекомендации 🟢/🔴 (общий блок по порогам).
5. Методология (Double Diamond, ICE, анти-галлюцинация).
6. Воронка конверсии.
7. B2B-пайплайн (если есть сделки).
8. **Анализ по каналам (детальный) — с графиком `channelBar`** + блок 🟢/🔴 по диаграмме.
9. **Воронка: визит → заявка → оплата — с графиком `funnel`** + блок 🟢/🔴.
10. **Анализ по каналам — с графиком `channelMix`** + блок 🟢/🔴.
11. Приоритизация гипотез (ICE) и карточки гипотез (если заведены).
12. AI-гипотезы и AI-решения (если сгенерированы).
13. Decision Log (обзор + детали).
14. AI-анализ (если есть нарратив).
15. Приложения: топ UTM / гео+устройства / страницы входа / выхода; дорожная карта; глоссарий;
    приложение с данными (прослеживаемость до `raw_responses`).

## Графики (FINAL §6.4)

| id           | Тип       | Данные снапшота                                   |
| ------------ | --------- | ------------------------------------------------- |
| `channelBar` | bar       | визиты + заявки по каналам (агрегат по имени)     |
| `funnel`     | funnel    | визиты → заявки B2C → B2B в работе → оплачено B2B |
| `channelMix` | pie/donut | доля каналов по визитам                           |

- Опции — `reportChartOption(snapshot, id)` в `@pca/shared` (чисто, тестируемо, 100%).
- Рендер PNG — `code/backend/src/report/chart-renderer.ts` (Puppeteer + ECharts UMD из workspace
  фронтенда). **Вне coverage** (как `pdf/renderer.ts`); проверяется e2e + вручную.
- DOCX встраивает PNG через `ImageRun`; PDF — через `<img src="data:image/png;base64,…">`.
- Графики рендерятся один раз на запрос генерации и передаются и в DOCX, и в PDF.

## Блок 🟢/🔴 у каждого графика (FINAL §6.3)

`chartRecommendation(snapshot, id)` (в `@pca/shared`, тестируемо) даёт детерминированную
светофор-интерпретацию **по каждой диаграмме**, привязанную к её разделу, по порогам
`REPORT_BENCHMARKS` (CR ≥5% / <2%, отказы ≥70%, доминирование канала ≥70%).

## ГОСТ-оформление

- A4, поля: левое 30 мм, правое 15 мм, верх/низ 20 мм.
- Times New Roman 14pt, межстрочный 1.5; заголовки разделов 14pt bold; нумерация страниц в колонтитуле.
- Таблицы с рамками; титульная и оглавление — отдельными страницами.

## Тестирование

- Опции и блоки 🟢/🔴 — unit (100%) в `@pca/shared`.
- Встраивание изображений — unit на `buildDocx`/`reportHtml` (PNG-фикстура): DOCX становится больше,
  PDF содержит `data:image/png;base64`.
- Рендер браузером — e2e/ручная проверка (DOCX в Word: изображения графиков + блок 🟢/🔴 после каждого).
