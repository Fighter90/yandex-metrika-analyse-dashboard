# Skill — Hypothesis Check

> Структурирует сырую гипотезу + считает ICE.

---

## Назначение

Превращает «у меня есть идея» в проверяемую гипотезу с критериями успеха и приоритетом.

## Промпт

```
Я хочу проверить продуктовую гипотезу. Помоги структурировать.

Сырая идея: {опишите идею своими словами}

Сделай следующее:

1. Переформулируй в формат:
   «{ЦА} {готова сделать что-то / готова заплатить за / переключится на} {решение}, если {условие}»

2. Найди скрытые допущения (минимум 3):
   - что мы предполагаем по поведению ЦА
   - что мы предполагаем по рынку
   - что мы предполагаем по технологии

3. Определи как проверить (минимум 2 способа):
   - синтетический CustDev → какие вопросы задать
   - живой CustDev → сколько респондентов и какие критерии
   - количественный → какой запрос к данным
   - рыночный → какие сигналы искать

4. Оцени по ICE (1–10), обоснуй каждую цифру в одну строку:
   - Impact: насколько изменит метрики, если подтвердится
   - Confidence: насколько уверен, что подтвердится
   - Ease: насколько легко проверить

5. Дай критерии:
   - Зелёный свет (продолжаем)
   - Жёлтый (pivot)
   - Красный (стоп)

6. Предложи дедлайн проверки (в днях).
```

## Что НЕ делать

- Не запускайте проверку без чёткого критерия успеха
- Не доверяйте ICE от AI без своей корректировки — это draft, не вердикт
- Не сравнивайте ICE между гипотезами разного масштаба

## Пример заполнения

| Поле | Значение |
|---|---|
| Сырая идея | Команды разметки готовы платить за ускорение постановки задач |
| Структура | Head of Annotation в командах ≥5 человек готов платить ₽30k/мес за инструмент, который сократит постановку задач с 40 мин до 10 мин, если результат сопоставим по качеству |
| Допущение 1 | Постановка задач — реальный bottleneck (не выдуманный) |
| Допущение 2 | Качество автогенерируемой инструкции достаточно для разметчика |
| Допущение 3 | 30k/мес — порог, до которого решают без CFO |
| Проверка качественная | 5 интервью с Head of Annotation за неделю |
| Проверка количественная | SQL по нашим теплым лидам: сколько уже отвечали что тратят >2ч/нед |
| ICE | I=8, C=5, E=7 → 280 |
| Зелёный | 3 из 5 респондентов: «да, готов платить ≥20k» |
| Жёлтый | 1-2 из 5: пересобрать УТП |
| Красный | 0 из 5 или возражения по качеству |
| Дедлайн | 10 дней |

## Доменные примеры — ProductCamp «Конверсии и лидген»

> ICE считается как **product** (I × C × E, диапазон 1–1000). См. `docs/methodology-ice.md`.

### Problem-гипотеза (фаза Define)

| Поле | Значение |
|---|---|
| Сырая идея | Подкаст даёт много заявок, но они плохо доходят до оплаты |
| Структура | Посетитель с подкаст-трафика **готов оставить заявку**, но не оплачивает в тот же день, **если** оплата требует ручного подтверждения реквизитов |
| Допущение (behavior) | Слушатель подкаста принимает решение об оплате не сразу, а «подумав» |
| Допущение (market) | Конкурирующие конференции дают мгновенную онлайн-оплату картой |
| Допущение (tech) | В Метрике корректно размечена цель «оплата» (id ≥ порога архивации) |
| Проверка (quantitative) | `channel_stats`: conversion-to-paid по `channel='podcast'` за 14 дней vs прямой трафик |
| Проверка (synthetic) | Synthetic CustDev: слушатель подкаста — что останавливает от оплаты сейчас |
| ICE | I=8, C=6, E=8 → **384** |
| 🟢 Зелёный | conversion-to-paid подкаста ниже прямого трафика на ≥30% (подтверждает разрыв) |
| 🟡 Жёлтый | разрыв 10–30% — нужен ещё срез по landing-страницам |
| 🔴 Красный | разрыва нет — гипотеза отклоняется |
| Дедлайн | 5 дней |

### Solution-гипотеза (фаза Develop, parent → problem выше)

| Поле | Значение |
|---|---|
| Структура | Посетитель с подкаст-трафика **переключится на** оплату онлайн-картой, **если** на новой посадочной «билет + проживание» добавить кнопку мгновенной оплаты |
| Допущение (behavior) | Барьер именно в способе оплаты, а не в цене |
| Допущение (market) | Картой готовы платить ≥60% B2C-аудитории |
| Допущение (tech) | Платёжный провайдер интегрируется за ≤3 дня |
| Проверка (live) | 5 интервью со слушателями, оставившими заявку, но не оплатившими |
| Проверка (quantitative) | A/B: доля оплат на новой посадочной vs старой за 7 дней |
| ICE | I=9, C=5, E=6 → **270** |
| 🟢 Зелёный | доля оплат на новой посадочной ≥1.5× старой за 7 дней |
| 🟡 Жёлтый | рост 1.1–1.5× — продлить тест |
| 🔴 Красный | без изменений — искать другой барьер |
| Дедлайн | 7 дней |
