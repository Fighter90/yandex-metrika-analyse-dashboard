📊 **ProductCamp Analytics — локальный дашборд для трека «Конверсии и лидген»**

Собрали инструмент, который за один `./run.sh` показывает, какой трафик приносит **оплаты**, а какой — пустые заявки. Все цифры тянутся из Яндекс.Метрики, кэшируются в SQLite и прослеживаются до исходного API.

**Что внутри:**
• 9 страниц: Обзор · Трафик · Поведение · Воронка · Цели · Отчёт · История · Настройки · Справка
• KPI к цели **300 платных билетов**: оплаты Метрики + B2B paid, gap, прогресс-ring
• Разбивка по каналам, UTM-Sankey (источник → кампания → заявки), гео, устройства, страницы входа
• Дайджест WoW, слабые места с deep-link «→ гипотеза», блоки 🟢/🔴/💡 под каждым графиком
• **AI-анализ** (Anthropic Claude) — 6 секций нарратива: Итог / Каналы / Воронка / Риски / Гипотезы / Вывод
• Генерация **DOCX и PDF по ГОСТ Р 7.32-2017** из неизменяемого среза — байт-в-байт по контенту
• «Что-если»-симулятор: сколько оплат при росте трафика и CR
• История снапшотов — любой старый отчёт открывается без перегенерации

**Для чего:**
Перестать вручную выгружать Excel'и. Видеть, какой канал реально приносит деньги, а не просто визиты. Формулировать гипотезы по методологии (формат + допущения + методы + ICE + светофор), фиксировать решения в Decision Log и обновлять приоритеты по факту.

**Как запустить:**

1. `git clone …`
2. `./init.sh` — OAuth Яндекс.Метрики + Anthropic ключ
3. `./run.sh` — миграции, sync, дашборд на `http://localhost:5173`

Версия **v2.9.3** — production-ready. Числа на Обзоре, Целях, в снапшоте и DOCX/PDF совпадают побайтово. Тесты 100%, регресс в браузере 93/94 ✓.
