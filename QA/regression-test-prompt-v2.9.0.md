## 0. Контекст продукта

Локальный инструмент (`./run.sh`): тянет Яндекс.Метрику (счётчик `<COUNTER_ID>` из `.env`) в SQLite,
показывает дашборд (frontend `:5173`, API `:4000`), ведёт гипотезы по методологии и Decision Log,
генерит детерминированные DOCX/PDF-отчёты по ГОСТ Р 7.32-2017 и опциональный AI-анализ (Anthropic).

Дашборд — **9 страниц**: Обзор, Трафик, Поведение, Воронка, Цели, Отчёт, История, Настройки, Справка.

### Ключевые инварианты (проверять в КАЖДОМ сценарии)

0. **Согласованность чисел между страницами и форматами:** одна метрика (визиты, заявки, CR) за один
   период совпадает на Overview / Traffic / Behavior / Funnel / Goals / Snapshot / превью / DOCX / PDF.
   Несовпадение = **Blocker** (нарушение anti-hallucination).
1. **Anti-hallucination:** любое число прослеживается до `raw_responses` в SQLite. Выдуманных нет.
2. **Заявка ≠ оплата:** заявки B2C (goal reaches) и оплаченные B2B-билеты разделены, не суммируются.
3. **Детерминизм отчёта:** один `snapshotId` → идентичный контент DOCX/PDF/экрана; в render-пути нет
   `Date.now()` и LLM (AI-нарратив генерится заранее и хранится в срезе); графики-PNG — побайтово
   идентичны при повторной генерации того же `snapshotId`.
4. **Гипотеза без полного формата методологии не сохраняется** (≥3 допущения по 3 категориям,
   ≥2 метода, светофор 🟢/🟡/🔴, дедлайн, ICE = I×C×E).
5. **Фильтры ≤ 1 года:** кастомный период ограничен 365 днями; пресеты 7д/14д/30д/90д/1г.
6. **KPI-прогресс = оплаты:** B2B paid **+** достижения цели-покупки (e_purchase) из Метрики; для
   цели-заявки — только B2B paid.
7. **UTM-покрытие** считается **визит-взвешенно** из `utm_stats` (помеченные визиты / все визиты),
   НЕ по доле строк channel_stats. На живых данных ≈ десятки %, НЕ 0%.

### Реалии живых данных (не дефекты)

- `GOAL_ID` может указывать на цель-**покупку** (`e_purchase`) — тогда `goal_reaches` = оплаты B2C;
  KPI-метка «Оплат». При `GOAL_ID=0` цель определяется автоматически.
- **Страницы выхода** (`ym:s:exitURL`) часто пусты — Stat API не отдаёт; блок скрывается.
- `bounceRate`/`conversionRate` хранятся как доля 0–1.

## 2. Матрица API (`:4000`, Swagger `/docs`)

- [ ] `GET /api/health` → 200, `{status:"ok", counterId, metrikaTokenPresent}`.
- [ ] `GET /api/settings` → значения маскированы (`****`), COUNTER_ID не маскирован.
- [ ] `GET /api/metrics/primary-goal` → объект цели (`type`, `name`); для ProductCamp `e_purchase`.
- [ ] `GET /api/metrics/goals` → список целей (активные + архивные при флаге).
- [ ] `GET /api/metrics/channels?from=&to=` → массив каналов; суммы совпадают с дашбордом.
- [ ] `GET /api/metrics/utm|geo-device|pages|exit-pages` → разбивки.
- [ ] `GET /api/hypotheses`, `GET /api/decisions`, `GET /api/b2b` → массивы.
- [ ] `POST /api/report/snapshot {from,to}` → снапшот с `id`, `kpi`, `goalLabel`, `funnel`, `b2bSummary`.
- [ ] `GET /api/report/snapshots` → список; `GET /api/report/snapshot/:id` → снапшот.
- [ ] `POST /api/report/generate {snapshotId,format}` → `{filePath}`; затем
      `GET /api/report/download/:snapshotId/:format` → байты (DOCX/PDF).
- [ ] `POST /api/report/insights {snapshotId}` → AI-нарратив (нужен ANTHROPIC_API_KEY).
- [ ] `POST /api/settings/refresh` → запускает синхронизацию (200).

## 6. 🔬 ГЛУБОКАЯ проверка DOCX (по содержимому)

Сгенерировать и распаковать (заменить SNAP на реальный id):

```bash
SNAP=$(curl -s -X POST localhost:4000/api/report/snapshot -H 'content-type: application/json' \
  -d '{"from":"2026-05-13","to":"2026-05-27"}' | python3 -c 'import sys,json;print(json.load(sys.stdin)["id"])')
curl -s -X POST localhost:4000/api/report/generate -H 'content-type: application/json' \
  -d "{\"snapshotId\":\"$SNAP\",\"format\":\"docx\"}" >/dev/null
curl -s "localhost:4000/api/report/download/$SNAP/docx" -o /tmp/r.docx
mkdir -p /tmp/rdocx && (cd /tmp/rdocx && unzip -qo /tmp/r.docx)
DOC=/tmp/rdocx/word/document.xml
```

- [ ] **Валидность:** `head -c2 /tmp/r.docx` = `PK`; размер разумный (≈120–250 КБ с графиками).
- [ ] **Титульная страница по ГОСТ:** организация «ProductCamp», трек «Конверсии и лидген», название
      работы, период, идентификатор среза, год — по центру, отдельной страницей.
- [ ] **«Содержание»** (TOC) отдельной страницей с нумерованными разделами.
- [ ] **Сквозная нумерация разделов** единая (1, 2, 3…); **нет** двойной нумерации вида
      «11. 1. Где мы сейчас» (AI-нарратив без собственной нумерации).
- [ ] **Нет дублирующихся секций:** `grep -o "Каналы: трафик и конверсия\|Каналы: структура трафика\|Анализ
по каналам (детальный)" $DOC | sort | uniq -c` — первые две присутствуют (по 2: TOC + заголовок),
      «Анализ по каналам (детальный)» = **0**.
- [ ] **Нет протечек markdown:** `grep -oE "####|#####|&lt;p&gt;|&lt;br" $DOC` — **пусто**.
- [ ] **3 PNG-графика встроены:** `ls /tmp/rdocx/word/media/` = 3 файла (бар каналов, воронка, микс);
      имена = SHA1-хэши контента (детерминизм).
- [ ] **Блок 🟢/🔴 у каждого графика** (интерпретация по порогам).
- [ ] **Приложение компактное:** `grep -oE "2026-[0-9-]{8} · " $DOC | wc -l` = **0** (нет посуточного
      построчного дампа); вместо него агрегированные итоги по каналам + ссылка на `raw_responses`.
- [ ] **Форматирование ГОСТ:** Times New Roman 14pt, межстрочный 1.5, поля 30/15/20/20 мм, нумерация
      страниц внизу, абзацы основного текста **по ширине** (justified), нет пустых блоков-заполнителей
      (не более одного пустого абзаца подряд).
- [ ] **Открыть в Word/LibreOffice** и визуально подтвердить вёрстку (титул, поля, выравнивание).

## 8. 🔬 AI-анализ (контент 6 секций)

Секции: **Итог · Каналы, UTM и Аудитория · Страницы и Воронка · Риски и Рекомендации ·

- [ ] AI **не нумерует собственные подзаголовки** — сквозную нумерацию даёт отчёт.
- [ ] Все числа в нарративе соответствуют снапшоту (anti-hallucination); «заявка ≠ оплата» соблюдён.
- [ ] «Итоговый вывод» — связный управленческий синтез: вердикт 🟢/🟡/🔴 по достижимости 300 +
      приоритетные следующие шаги.
- [ ] Повторная генерация для того же снапшота заблокирована (кнопка disabled).

## 13. Формат фиксации дефектов

Для каждого дефекта: **ID · Приоритет · Где (URL/файл/раздел) · Шаги · Ожидаемо · Фактически · Док-привязка.**

Приоритеты: 🔴 **Blocker** (нарушает инвариант 0–7 или красный автогейт) · 🟠 **Critical** ·
🟡 **Major** · 🟢 **Minor**.

**Светофор готовности:**

- 🟢 **GO** — автогейт зелёный; 0 Blocker/Critical; DOCX/PDF проходят раздел 6–9; консоль чистая.
- 🟡 **CONDITIONAL** — есть Major, но базовые сценарии и отчёты работают.
- 🔴 **NO-GO** — есть Blocker/Critical или красный автогейт.
