## 🔴 Открытые дефекты

### 🟡 M-004 [Major] — UTM-coverage всё ещё показывает 0.0%

- **Где:** /traffic и /overview бейдж «Низкое покрытие UTM: 0% (порог 70%)».
- **Контекст:** заявлено в release notes v2.6.0+, но в v2.8.3 значение по-прежнему 0%. Возможно расчёт идёт не из `utm_stats`, а из `channel_stats.utmCoverage`.
- **Не блокирует пилот**, но снижает доверие к одному из чисел.

### 🟢 m-charts-snapshot [Minor] — `snapshot.charts` undefined в API ответе

- **Где:** `GET /api/report/snapshot/:id` — поле `charts` не возвращается.
- **Контекст:** DOCX содержит 3 PNG-файла со SHA1-именами, значит рендер происходит на лету при экспорте, либо хранятся в файловой системе (не в JSON). Это OK для детерминизма (имена-хэши гарантируют идентичность), но не оптимально для prefetch UI.

### ⚪ Не успел проверить

- **Live AI insights** генерация (не запускал, чтобы не тратить токены без согласия) — заявлено 6 секций включая «Итоговый вывод», HTML без MD-тегов.
- **Sankey разворот на весь экран** на /traffic — кнопка «⛶ Развернуть» не тестировал клик.
- **DOCX/PDF offline open** в Word/LibreOffice — проверка ГОСТ-титула, line-spacing 1.5, полей 30/15/20/20мм требует GUI.
- **Mobile pass 375×812** — Chrome MCP не сжимает viewport.
- **axe-core a11y** — нужен headless run.
- **SHA-256 идентичность DOCX/PDF двух генераций одного snapshotId** — PNG имена-хэши уже доказывают определённый детерминизм.
- **Live sync re-run** — БД свежая, не запускал.
