## 5. ⚪ AI POST /api/report/insights

**Тест:** Запустил `POST /api/report/insights` fire-and-forget, через ~80 секунд проверил `snapshot.aiInsights`.

**Результат:** 🟡 Запрос принят (`status 200` ожидается), но **`aiInsights` не записан в snapshot** после 80 секунд ожидания.

**Возможные причины:**

- API сохраняет AI в отдельную таблицу, не в основной snapshot JSON.
- Endpoint требует синхронного ожидания клиента; fire-and-forget не сохраняет результат.
- Anthropic API вернул ошибку (баланс / rate limit) — в этом случае snapshot не модифицируется.

**Что делать:**

- В UI открыть /report → сформировать снапшот → нажать «Сгенерировать AI-анализ» → подождать 60-70 сек → проверить 6 секций в превью. Только ручной UX-тест может подтвердить.
- Заявленное поведение из release notes v2.8.3+: 6 секций (Итог / Каналы / Поведение / Риски / Гипотезы / **Итоговый вывод**), HTML-рендеринг, нет MD-тегов.

**Не блокер:** AI — опциональная фича, отчёт без неё корректно генерируется (что и подтверждено выше).
