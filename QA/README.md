# QA — промпты для тестирования

Подробные, самодостаточные промпты для всесторонней проверки продукта. Каждый можно отдать человеку
или AI-агенту с доступом к репозиторию и запущенному приложению (`./setup.sh`).

> 🔄 Актуально для **v0.9.0**: 11 страниц, авто-определение KPI-цели, детальный ≥25-стр. отчёт
> (на экране + DOCX/PDF), опциональный AI-анализ.

| Файл                                                         | Роль                   | Назначение                                                                                                                                                                    |
| ------------------------------------------------------------ | ---------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [regression-test-prompt.md](regression-test-prompt.md)       | QA-инженер             | Полный регресс всего функционала: парсер, API, 11 страниц, отчёты, AI, граничные случаи + инварианты (anti-hallucination, «заявка ≠ оплата», детерминизм, формат Воронковой). |
| [ux-analyst-prompt.md](ux-analyst-prompt.md)                 | UX-аналитик            | Решает ли продукт задачи ЦА; разрывы «поведение ↔ потребности»; онбординг волонтёра; доверие к данным.                                                                        |
| [ux-designer-prompt.md](ux-designer-prompt.md)               | UX-проектировщик       | Информационная архитектура, навигация, потоки, состояния экранов, формы, когнитивная нагрузка.                                                                                |
| [ux-visual-designer-prompt.md](ux-visual-designer-prompt.md) | UX/визуальный дизайнер | Типографика, цвет/семантика, сетка, графики ECharts, доступность WCAG AA, наглядность.                                                                                        |

Рекомендованный порядок прогона: **регресс** (работает ли всё) → **UX-аналитик** (полезно ли) →
**UX-проектировщик** (удобно ли структурно) → **визуальный дизайнер** (понятно ли визуально).

Критерии приёмки для новых фич берутся из их спецификаций — см.
[`../docs/specs/`](../docs/specs/README.md) (Spec-Driven Development). Регресс-промпт проверяет в т.ч.
эти критерии после реализации спеки.
