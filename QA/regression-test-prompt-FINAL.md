## 0. Контекст и сквозные инварианты

Локальный инструмент (`./run.sh`): тянет Яндекс.Метрику (`<COUNTER_ID>` из `.env`) в SQLite,
показывает дашборд (frontend `:5173`, API `:4000`, Swagger `/docs`), ведёт гипотезы по методологии и
Decision Log, генерит детерминированные DOCX/PDF по ГОСТ Р 7.32-2017 и опциональный AI-анализ (Anthropic).
Дашборд — **9 страниц**: Обзор, Трафик, Поведение, Воронка, Цели, Отчёт, История, Настройки, Справка.

**Инварианты (проверять в каждом сценарии):**

0. **Согласованность чисел** между страницами и форматами (Overview/Traffic/Behavior/Funnel/Goals/
   снапшот/превью/DOCX/PDF) за один период — совпадают. Несовпадение = **Blocker**.
1. **Anti-hallucination:** любое число прослеживается до `raw_responses`. Выдуманных нет.
2. **Заявка ≠ оплата:** заявки B2C (goal reaches) и оплаченные B2B-билеты разделены, не суммируются.
3. **Детерминизм (content):** один `snapshotId` → идентичные `word/document.xml` + PNG (имена = SHA1).
   Файловый SHA-256 контейнера может отличаться (zip/pdf-таймстемпы) — **known limitation**, не дефект.
4. **Формат гипотезы обязателен:** ≥3 допущения (3 категории), ≥2 метода, светофор 🟢/🟡/🔴, дедлайн,
   ICE = I×C×E (произведение).
5. **Фильтры ≤ 1 года:** кастомный период ≤ 365 дней; пресеты 7д/14д/30д/90д/1г.
6. **KPI-прогресс = оплаты:** B2B paid **+** достижения цели-покупки (e_purchase) из Метрики; для
   цели-заявки — только B2B paid.
7. **UTM-покрытие** — визит-взвешенно из `utm_stats` (помеченные визиты / все визиты), НЕ доля строк
   channel_stats. На живых данных ≈ десятки %, НЕ 0%.

**Реалии живых данных (не дефекты):** цель может быть e_purchase (метка «Оплат»); страницы выхода
(`ym:s:exitURL`) часто пусты; `bounceRate`/`conversionRate` хранятся как доля 0–1.

## 7. 🔬 DOCX по содержимому

```bash
SNAP=$(curl -s -X POST localhost:4000/api/report/snapshot -H 'content-type: application/json' \
  -d '{"from":"2026-05-13","to":"2026-05-27"}' | python3 -c 'import sys,json;print(json.load(sys.stdin)["id"])')
curl -s -X POST localhost:4000/api/report/generate -H 'content-type: application/json' \
  -d "{\"snapshotId\":\"$SNAP\",\"format\":\"docx\"}" >/dev/null
curl -s "localhost:4000/api/report/download/$SNAP/docx" -o /tmp/r.docx
mkdir -p /tmp/rdocx && (cd /tmp/rdocx && unzip -qo /tmp/r.docx); DOC=/tmp/rdocx/word/document.xml
```

- [ ] Валидный ZIP (`head -c2 /tmp/r.docx` = `PK`), размер ≈ 120–250 КБ с графиками.
- [ ] **Титул по ГОСТ:** организация, трек, название работы, период, ИД среза, дата, цель, год — по
      центру, отдельной страницей. **Нет 2+ пустых абзацев подряд** (D-EMPTY-PAGE).
- [ ] **«Содержание»** (TOC) с нумерованными разделами.
- [ ] **Нет дублей секций по каналам:** `grep -o "Каналы: трафик и конверсия\|Каналы: структура трафика\|Анализ по каналам (детальный)" $DOC | sort | uniq -c` → первые две есть, третья = 0.
- [ ] **Нет markdown-протечек:** `grep -oE "####|<p>|<br" $DOC` → пусто.
- [ ] **Нет двойной нумерации:** `grep -oE ">[0-9]+\. [0-9]+\." $DOC` → пусто (D-DOUBLE-NUM).
- [ ] **3 PNG-графика:** `ls /tmp/rdocx/word/media/` = 3 (имена = SHA1). Блок 🟢/🔴 у каждого.
- [ ] **Приложение компактное:** `grep -oE "2026-[0-9-]{8} · " $DOC | wc -l` → 0 (агрегаты по каналам).
- [ ] **ГОСТ-формат:** Times New Roman 14pt, межстрочный 1.5, поля 30/15/20/20 мм, нумерация страниц,
      абзацы по ширине (justified).

## 17. Форма фиксации дефектов и светофор

Дефект: **ID · Приоритет · Где · Шаги · Ожидаемо · Фактически.**
Приоритеты: 🔴 **Blocker** (нарушает инвариант 0–7 или красный автогейт) · 🟠 **Critical** · 🟡 **Major** · 🟢 **Minor**.

**Светофор готовности:**

- 🟢 **GO** — автогейт зелёный; §2–§13 пройдены; §14 подтверждён отдельно; 0 Blocker/Critical.
- 🟡 **CONDITIONAL** — есть Major; базовые сценарии и отчёты работают; §14 не весь закрыт.
- 🔴 **NO-GO** — есть Blocker/Critical или красный автогейт.
