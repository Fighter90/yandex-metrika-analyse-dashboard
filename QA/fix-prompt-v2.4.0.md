## 2. Stepper-форма «Новая гипотеза» (P0)

### Спецификация (`docs/specs/011-hypothesis-form-stepper.md`)

5 шагов (см. `ux-designer-analysis-v2.3.0.md §5`):

1. **Сформулировать** — шаблон фразы Воронковой с dropdown «готова сделать / готова заплатить за / переключится на».
2. **Допущения** — ≥3 в 3 категориях (риск/гипотеза/неизвестное).
3. **Методы** — ≥2 метода с метрикой и сроком.
4. **ICE** — три слайдера 1–5 с обоснованием каждой оценки, итог = I×C×E (не среднее, см. ADR-005).
5. **Светофор + дедлайн + привязка snapshot**.

### Технические требования

- Библиотека: `react-hook-form` + `zod`-схема для валидации.
- Inline-валидация на каждом поле + on-blur.
- Кнопка «Сохранить как DRAFT» на каждом шаге (не теряется работа).
- Прогресс-бар (5 точек) фиксированный сверху.
- При завершении — POST `/api/hypotheses` с полем `status: 'draft' | 'active'`.
- Подсказки из `.claude/skills/hypothesis-check/SKILL.md` — отдельный компонент `<HypothesisHint slot="step1">…</HypothesisHint>`.

### Тесты

- Unit: zod-схема отвергает гипотезу без 3 допущений / 2 методов / светофора / ICE.
- Functional: API создаёт гипотезу со статусом draft.
- E2E: пользователь проходит 5 шагов, сохраняет, видит на /hypotheses в списке.

- На /hypotheses/new — `useSearchParams()` распарсить, передать в form defaults.

## 5. Глобальный header-badge источника данных (P1)

### Спецификация

Под топ-навигацией — sticky-полоса:

```
🟢 Live · Метрика 54280963 · synced 27 мая 03:37 · период 14–27 мая · сегмент B2C+B2B  [↻ Refresh]
```

Цвет бейджа по правилу:

- 🟢 sync < 1 ч.
- 🟡 1–24 ч → подпись «Обновить?»
- 🔴 > 24 ч → «Данные устарели — обновите перед презентацией».
- ⚪ `COUNTER_ID=0` → большой жёлтый бейдж «DEMO».

### Технически

- `GET /api/health` уже отдаёт `counterId` и `timestamp`; нужен ещё `lastSyncAt` в health (читать `MAX(synced_at)` из `channel_stats`).
- Компонент `<DataSourceBadge>` в Layout, обновляется каждые 60с (react-query refetchInterval).
- На /help и /settings бейдж скрыт (как FilterBar).

## 12. Метрики usability приёмки v2.5.0

В дополнение к §8.13 fix-prompt-v2.3.0 (анти-галлюцинация, ГОСТ, etc.) — usability-метрики:

| Метрика                                     | Целевое значение    | Как измеряем                                                           |
| ------------------------------------------- | ------------------- | ---------------------------------------------------------------------- |
| Time-to-first-hypothesis (новый волонтёр)   | ≤ 10 минут          | таймер на онбординг-туре + создание первой гипотезы                    |
| Кликов от Overview до созданной гипотезы    | ≤ 3                 | трассировка `pca.deep_link.click` → `pca.hypothesis.create`            |
| Кликов от снапшота до экспортированного PDF | ≤ 4                 | `pca.report.generate` → `pca.export.click`                             |
| Понимание KPI пользователем (survey)        | ≥ 4 из 5            | внутренний опрос команды                                               |
| Lighthouse Accessibility                    | ≥ 90 на 9 страницах | CI: `lighthouse-ci`                                                    |
| Mobile Lighthouse Performance               | ≥ 75                | то же                                                                  |
| Пустые состояния показаны                   | 100%                | E2E: на каждой странице с `?empty=1` query — `<EmptyState>` рендерится |
| Контекст фильтра сохраняется                | да                  | E2E: фильтр на /overview сохраняется при переходе на /report           |
