# Skill — Market Scan (ProductCamp adaptation)

> Превращает «погугли мне рынок» в структурированный запрос с источниками.

---

## Назначение

Получить картину рынка продуктовых конференций / каналов привлечения за 15 минут вместо 4 часов. С источниками — иначе это галлюцинация.

В контексте ProductCamp — используется в фазе **Develop** Double Diamond, когда нужно понять, как конкуренты решают проблему, которую мы определили на фазе Define.

## Инструмент

Любой из:
- Qwen с web search / research-lookup skill (рекомендуется — встроен в среду)
- Perplexity (платно, лучше всего с источниками)
- EXA, Tavily — для встраивания в скрипты

## Промпт-шаблон

```
Сделай market scan по теме: {точное описание ниши}

Контекст: я работаю над треком конверсии для платной офлайн-конференции
для продакт-менеджеров (ProductCamp). KPI — 300 платных билетов.
Меня интересует {конкретный аспект}.

Структура отчёта:

1. Размер рынка
   - Глобально (TAM продуктовых конференций) — если есть
   - В РФ/СНГ (мой регион)
   - Темп роста за последние 2 года
   - Источник для каждой цифры (без источника — не включай)

2. Игроки (минимум 5 прямых или косвенных конкурентов):
   - Название конференции/инструмента
   - Продукт (одной строкой)
   - ICP (для кого)
   - Ценник (от/до, модель — early-bird, group rate, B2B-пакеты)
   - Сильные стороны (2)
   - Слабые стороны (2)
   - Источник информации

3. Тренды (3):
   - Что меняется в продуктовых конференциях/обучении
   - Кто двигает изменение
   - Сигнал для ProductCamp

4. Регуляторные риски:
   - Есть ли законы/стандарты, которые влияют (НДФЛ на обучение от компании, эквайринг для физ.лиц, и т.п.)

5. Белые пятна (2):
   - Что прямые конкуренты НЕ закрывают
   - Где есть незанятая ниша для ProductCamp

Правила:
- Каждая цифра — с источником. Без ссылки — не включай.
- Если данных нет — напиши «данных не нашёл», не выдумывай.
- Используй данные не старше 18 месяцев. Старше — помечай как историческое.
- Региональные данные (RU/CIS) часто бедные — закладывай это.
```

## Темы market-scan, актуальные для нашего трека

Используй для гипотез решений, чтобы не изобретать велосипед:

1. **Промежуточные лендинги между подкастом и оплатой** — как другие подкасты конвертируют слушателей в платных участников событий.
2. **B2B-пакеты для конференций** — как продают «команда на 10 человек» (структура offer'а, цены, лендинги).
3. **Early-bird / Late-bird ценообразование** — реальные примеры и эффект на conversion (Conf42, ICAIMT, ProductSense, и т.п.).
4. **Программы амбассадоров** — как другие конференции мотивируют рефералов: процент, фиксированный бонус, доступ.
5. **UTM-разметка в подкастах** — best practices, как другие подкасты-конференции отслеживают эффективность эпизодов.
6. **Конверсия продакт-лендингов** — бенчмарки conversion rate (визит → оплата) для русскоязычных продуктовых событий.

## После получения отчёта — проверочные вопросы

```
По этому отчёту:
1. Какие из утверждений я бы перепроверила вручную?
2. Какие источники могут быть предвзятыми (паблики конференций, маркетинговые материалы)?
3. Что в отчёте противоречит моему опыту / опыту команды? Почему?
4. Какие 3 вопроса я задам реальному человеку из этой индустрии (например, организатору ProductSense)?
5. Какие 2 механики из конкурентов мы могли бы воспроизвести за 1 неделю?
```

## Метрика качества market-scan

Хороший market-scan = **отчёт, в котором есть минимум одна цифра, которая удивит**.
Если всё «как я и предполагал» — переформулируй на более узкий сегмент (например, не «продуктовые конференции», а «русскоязычные офлайн-конференции для продактов 2024–2025»).

## Ограничения

- AI оптимизирует под «полный отчёт», а не точность — выдаёт цифры даже когда их нет.
- Источники могут быть устаревшими — проверяй дату.
- Региональные данные (особенно RU/CIS, особенно по нишевым конференциям) бедные — закладывай 30–60 мин ручного поиска поверх AI.
- Цены конференций часто меняются (early-bird закончился, late-bird поднялся) — проверяй на сайтах напрямую.
